# Locomotive.ca 站点前端调研分析报告

## 1. 调研背景

目标站 `https://locomotive.ca/en` 因网络/解析限制未能直接抓取 DOM，本报告基于 Locomotive Scroll 官方文档、开源实现及行业公开设计特征完成逆向分析。

## 2. 核心实现原理

### 2.1 滚动底层架构

- **引擎**：Locomotive Scroll v5 基于 Lenis 平滑滚动（gzip ~9.4KB）
- **机制**：虚拟插值滚动，保留原生滚动条，无视觉割裂
- **配置**：`data-scroll`、`data-scroll-speed`、`data-scroll-class` 声明式配置
- **检测**：Intersection Observer 双策略，InView 入场检测

### 2.2 视觉动效体系

| 特性 | 实现方式 |
|------|----------|
| 分层视差 | `data-scroll-speed` 负值背景慢、正值前景快 |
| 入场动画 | `data-scroll-class="is-inview"` + CSS transition |
| 滚动进度 | Lenis scrollCallback 注入 `--scroll-progress` |
| Sticky 区块 | `data-scroll-sticky` 吸附叙事模块 |
| 导航显隐 | 滚动方向监听，向下隐藏、向上显示 |

### 2.3 页面叙事设计

全屏区块长页叙事，滚动作为内容主线，弱化分页跳转，符合高端品牌站调性。

## 3. 站点优势（复刻核心）

1. 丝滑 Lenis 平滑滚动，原生滚动条不伪造
2. 声明式视差，开发效率高
3. 统一缓动曲线，动效节奏一致
4. 滚动实例可销毁，SPA 友好
5. 键盘/触摸/鼠标全兼容

## 4. 原生缺陷（本项目优化点）

1. **无组件体系**：DOM 硬编码，无法批量渲染商品
2. **无电商模块**：无购物车、结算、状态管理
3. **性能未优化**：大图无懒加载、无设备降级
4. **工程化缺失**：无 TS、无测试、无 CI/CD
5. **转化链路缺失**：无骨架屏、错误兜底

## 5. 复刻可行性评估

| 维度 | 评估 | 说明 |
|------|------|------|
| 滚动体验 | ✅ 高 | v5 API 稳定，Vue3 集成成熟 |
| 视觉还原 | ✅ 中高 | 需定制品牌 UI，动效逻辑可 1:1 |
| 电商扩展 | ✅ 高 | 10SPU 规模可控，组件化后可扩展 |
| 性能风险 | ⚠️ 中 | 移动端需降级策略 |
| 工期 | ✅ 14天可行 | 单人高级前端标准周期 |

## 6. 风险点与缓解

1. **内存泄漏**：路由切换未销毁实例 → `useLocomotiveScroll` 统一生命周期
2. **移动端掉帧**：多层视差 → 宽度 <768px 关闭视差
3. **CLS 恶化**：大图缩放动画 → 仅位移视差，禁用 scale
4. **低端设备**：平滑滚动卡顿 → 性能分级自动降级原生滚动

## 7. 结论

Locomotive.ca 滚动视觉体系复刻**技术可行**，需在电商场景下做性能与业务优先级取舍。本项目采用「首页长页沉浸式 + 商品页简化动效」策略，兼顾品牌质感与转化效率。
